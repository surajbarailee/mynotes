Backup

A backup, or data backup is a copy of computer data taken and stored elsewhere so that it may be used to restore the original after a data loss event.

Backups can be used to recover data after its loss from dta deletion or corrution , or to recover data from an earlier time.Backups provide a simple form of disaster recovery however not all backup systems are able to reconstitute a computer system or other complex configuration such as a computer cluster,active directory server or database server .A backup system contains at least one copy of all data considered worth saving. The  data storage  requirements can be large.

 

Types of Backup:

Full Backup

  • Makes a copy of all data to another set of media
  • Run mostly periodically
  • Consumes more space
  • Example:pushing first time to github

Advantage:

  • Complete copy of all data is available with a single set of media.So minimal time to restore data

Disadvantage:

  • More storage
  • Time consuming

Incremental Backup

  • Copying only the data that has changed since the las backup operation of any type.
  • more frequent than full backup
  • Tracks file modification

Advantages:

  • Can be run more frequently
  • Copies smaller amount so faster speed

Disadvantage:

  • Time consuming to restore

 

Differential Backup

  • Similar to incremental backup But references the full backup for change comparision
  • Advantage over incremental is Better restore time

 

Mirror Backup

Mirror of the source being backed up.For eg:when you add a text "my device" to a text file,the back up file in the server is automatically updated.

Advantage:

  • Fastest backup type
  • More than one copy backup sites
  • Creates a snapshot of selected files without need of running seperate command

Disadvantage:

  • More storage space
  • MAy be incapable of version tracking.
