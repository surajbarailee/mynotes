FAT file system

A file allocation table (FAT) is a file system that originally used 12 or 16 bits for each cluster entry into the file allocation table. It is used by the operating system (OS) to manage files on hard drives and other computer systems, flash memory, digital cameras and portable devices.

Supports individual files up to 4GB in size and volumes up to 2TB in size.

FAT32 isn't a journaling file system, which means that file system corruption can happen

FAT32 doesn't support file permissions.

FAT just saves some table of contents so less resources are used

A drive can be only up to 8TB if using FAT32 (with some serious hacking). By default Windows disallows a FAT32 drive larger than 32GB.

File and folder names are actually limited to 8 characters with an extension of 3 characters.

FAT32 has serious problems with file names using non-English letters - to the point where you could find a file unable to be opened at all.

FAT12,FAT16, FAT32,exFAT are some of its types differering in cluster size.

exFAT is optimized for flash memories —designed to be a lightweight file system like FAT32, but without the extra features and over head of NTFS and without the limitations of FAT32. exFAT has very large limits on file and partition sizes compared to FAT32

Structure of FAT is given below

  • Boot Sector: This is also known as the reserved sector; it is located on the first part of the disc. It contains: the OS's necessary boot loader code to start a PC system, the partition table known as the master boot record (MRB) that describes how the drive is organized, and the BIOS parameter block (BPB) which describes the physical outline of the data storage volume.
  • FAT Region: This region generally encompasses two copies of the File Allocation Table which is for redundancy checking and specifies how the clusters are assigned.
  • Data Region: This is where the directory data and existing files are stored. It uses up the majority of the partition.
  • Root Directory Region: This region is a directory table that contains the information about the directories and files. It is used with FAT16 and FAT12 but not with other FAT file systems. It has a fixed maximum size that is configured when created. FAT32 usually stores the root directory in the data region so it can be expanded if needed.
