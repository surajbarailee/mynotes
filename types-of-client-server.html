Types of Client Server Architecture

In client server architecture ,client and server may reside on the same computer or in different computer conncted through a network.Server acts as the producer and the client as a consumer.Client Server Architecture can be classified according to the number of layer it has.It may be 1-tier, 2-tier, 3-tier .... and n-tier.Lets see a little more about it.

1-tier architecture

In this architecture ,the user interface/presentation ,marketing/application logic and data logic are present in the same system.One of the simple example can be a software package like Ms office.Another example could be using a localhost to serve a html page.

These kind of architecture is cheaper than others but also difficult to handle due to data inconsistency.

 

2-tier architecture

In this architecture,the user interface is stored in client system where as database are stored on server.Database logic and business logic are stored at either client or server.This architecture is useful when a client talks directly to a server.

If client side has BL and DL it is called as fat client thin server architecture.

If server stores BL and DL it is called thin client fat server architecture.

This architecture is affordable and easy to maintain.

 

3-tier architecture

As name suggest,it has three layers :user interface,middleware and database server.

Client's request goes to middle layer ,middle layer requests to server ,takes the response and forwards it to the client.It gives databse more isolation and security.These systems are expensive but simple to manage .

Client manages presentation layer,the application server takes care of the application layer where logics like what to give to client ,when to give etc resides and the erver system supervises database layer.For simplicity( A html page,php and mysql example can be taken)
